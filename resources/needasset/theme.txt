PURPOSE:

================================================================================
  SVG / ICON RECOLORING TABLE
================================================================================

  The application dynamically recolors SVG icons at runtime. These are the
  color replacement rules applied to all SVG assets.

  File: src/slic3r/GUI/BitmapCache.cpp (lines 327-342)

  LIGHT MODE ICON REPLACEMENTS:
    #0x00AE42  -->  #009688   (green icons become accent teal)
    #00FF00    -->  #52c7b8   (bright green becomes lighter teal)

  DARK MODE ICON REPLACEMENTS:
    #262E30  -->  #EFEFF0   (dark text becomes light text)
    #323A3D  -->  #B3B3B5   (secondary dark text lightens)
    #CECECE  -->  #54545B   (light borders become dark borders)
    #6B6B6B  -->  #818182   (muted text slightly lighter)
    #909090  -->  #FFFFFF   (dim gray becomes white)
    #009688  -->  #00675b   (accent teal darkens for contrast)
    #F1F1F1  -->  #36363B   (light bg becomes dark bg)
    #DBDBDB  -->  #4A4A51   (input border darkens)
    #F0F0F1  -->  #333337   (disabled bg darkens)

================================================================================
  REBRANDING INSTRUCTIONS
================================================================================

  MINIMUM VIABLE REBRAND (fastest):
    Replace #009688 globally with your new brand accent color.
    This covers ~80% of the visual identity change.

  FULL REBRAND:
    1. Choose a primary accent color and its dark mode variant
    2. Choose hover, pressed, and tint variants of the accent
    3. Fill in all REPLACEMENT fields above
    4. Dev team will do a find-and-replace across the source files listed
    5. CSS files can be updated with simple search-replace

  IMPORTANT:
    - Maintain sufficient contrast ratios (WCAG AA minimum)
    - Dark mode colors should be slightly desaturated versions
    - Test both light and dark themes after changes
    - SVG icon recoloring in BitmapCache.cpp must match new palette
